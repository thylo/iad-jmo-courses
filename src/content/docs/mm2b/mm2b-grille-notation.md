# Grille de notation — MM2B : Site web narratif pour enfants

## Critère 1 : Qualité du code HTML/CSS (40 points)

### Structure HTML (12 points)

- **Sémantique correcte** (5 pts) : Utilisation pertinente de `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>`, etc.
- **Indentation et lisibilité** (4 pts) : Code proprement indenté, facile à lire
- **Nommage des classes** (3 pts) : Noms explicites et cohérents (pas de `.box1`, `.truc`)

### Organisation CSS (12 points)

- **Fichier séparé** (2 pts) : CSS dans un fichier externe, pas de styles inline partout
- **Sélecteurs clairs** (4 pts) : Sélecteurs précis, pas de `.class .class .class .class`
- **Pas de répétitions** (3 pts) : Code DRY, pas de copier-coller de styles
- **Commentaires pertinents** (3 pts) : Sections importantes commentées

### Mise en page et responsive (16 points)

- **Flexbox ou Grid** (6 pts) : Utilisation correcte des deux
- **Responsive fonctionnel** (6 pts) : Le site fonctionne sur mobile et desktop
- **Typographie** (2 pts) : Hiérarchie visuelle claire
- **Palette de couleurs** (2 pts) : Cohérente et adaptée au public enfant



## Critère 2 : Utilisation des concepts du cours (30 points)

### Concepts techniques appliqués (20 points)

- **HTML sémantique maîtrisé** (5 pts) : Choix de balises justifiés et pertinents
- **Layout Flexbox/Grid** (6 pts) : Mise en page réfléchie et efficace
- **Media queries** (5 pts) : Responsive bien pensé, pas juste du `max-width: 768px` au hasard
- **Variables CSS** (4 pts) : Utilisation pour couleurs, espacements, etc.

### Capacité d'explication (10 points)

- **Justification des choix HTML** (3 pts) : Pourquoi cette balise plutôt qu'une autre ?
- **Explication du layout** (4 pts) : Comment fonctionne la mise en page CSS ?
- **Démarche responsive** (3 pts) : Pourquoi cette approche responsive ?



## Critère 3 : Originalité et intention narrative (20 points)

### Qualité narrative (10 points)

- **Histoire claire** (5 pts) : Le récit est compréhensible et cohérent
- **Adaptation au public** (5 pts) : Vocabulaire, ton, visuels adaptés aux 6-12 ans

### Univers visuel et expérience (10 points)

- **Cohérence visuelle** (4 pts) : Couleurs, typographie, images forment un tout
- **Réflexion UX enfant** (3 pts) : Navigation intuitive, clarté, engagement
- **Originalité** (3 pts) : Le projet se démarque, apporte quelque chose de personnel



## Critère 4 : Compréhension individuelle (10 points)

### Présentation orale (10 points)

- **Identification claire** (2 pts) : L'étudiant sait précisément ce qu'il a codé
- **Explication ligne par ligne** (5 pts) : Capacité à expliquer le code montré
- **Réponses aux questions** (3 pts) : Compréhension réelle, pas de flou



## Barème total : 100 points


## Pénalités possibles

- **Code non fonctionnel** (site ne s'affiche pas) : -15 points
- **Plagiat entre groupes** : 0/20 automatique
- **Code IA non compris** (évident lors de la présentation) : -10 points sur compréhension
- **Absence de participation** (un membre n'a rien fait) : note individuelle fortement réduite, pas de pénalité pour le reste du groupe
- **Absence de complémentarité** (répartition incohérente, pas de travail d'équipe réel) : -10 points

## Bonus possibles

- **Hébergement réussi** (site en ligne, URL fonctionnelle) : +2 points
- **Animations CSS pertinentes** (si elles servent le récit) : +2 points
- **Accessibilité pensée** (contrastes, alt sur images) : +2 points

Maximum de bonus cumulables : 5 points
